# ADR 0003：可控扩展运行时

- 状态：已接受
- 日期：2026-07-19

## 背景

Skills、项目规则、Hooks 和 MCP 都会影响模型上下文或增加可执行能力。如果分别直接接入智能体循环，会产生工具名覆盖、权限绕过、密钥泄露、配置错误降级放行和无法审计的问题。

## 决策

1. 新增独立 `extensions` 领域模块，负责发现、严格校验、生命周期和诊断；`agent` 只接收已经编译的运行时指令和 `ToolRegistry`。
2. 指令按“全局、项目根、项目规则”从低到高排序。Skill 使用严格 YAML frontmatter，项目同名 Skill 覆盖全局 Skill；触发后由运行时在 Provider 请求前读取正文。
3. 非只读 Skill 默认停用，必须由宿主启用。Skill 指令永远不能扩大工具权限。
4. MCP 使用 `mcp__<server>__<tool>` 命名空间；规范化后冲突或与内置工具重名时整个扩展加载失败。
5. MCP 工具转换为普通 `ToolHandler`，继续经过 Schema、`PolicyEngine`、一次性审批、取消、Hook、持久化工具结果和审计流程。
6. stdio 使用换行分隔的 UTF-8 JSON-RPC；Streamable HTTP 使用单端点 POST，并维护协议版本和会话 ID。当前支持 MCP `2025-11-25`。
7. MCP 凭据只存入独立操作系统凭据账户。配置文件只保存凭据名称到环境变量或 Header 的映射，诊断只返回“是否已配置”。
8. Hook 使用结构化命令，通过 stdin 接收有界 JSON，并必须返回 `{decision: allow|block}`。超时、非零退出、超限或格式错误都按阻止处理。
9. 配置、指令和 Skill 文件使用修订指纹。发现变化后重新构建完整注册表；失败时不保留“已准备”状态，后续 Turn 继续关闭失败。

## 影响

收益：

- 外部工具复用核心安全边界，不能静默替换内置工具。
- 密钥不进入模型上下文、工具参数、配置载荷或结构化日志。
- 格式错误和生命周期错误对 Turn 是确定性的关闭失败。
- 设置界面可以从运行时诊断和审计历史重建状态。

成本与限制：

- 当前不支持旧版 HTTP+SSE MCP 传输和实验性 Tasks。
- MCP 服务器请求的 sampling、elicitation 和 roots 暂不支持，并返回 JSON-RPC `Method not found`。
- Hook 是本地宿主能力，配置者必须信任其结构化命令；进程只接收最小环境变量集。

## 未采用方案

### 让 MCP 客户端绕过 ToolRegistry 直接执行

未采用，因为会绕过参数校验、审批、取消、Hooks 和审计。

### 把密钥写入 extensions.json

未采用，因为项目文件、日志、模型读取和版本控制都可能泄露密钥。

### 配置错误时跳过单个扩展

未采用，因为静默降级会让用户误判实际生效的规则和安全边界。
