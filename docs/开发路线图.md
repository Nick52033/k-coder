# k-Coder 开发路线图

本文档是项目开发顺序的唯一事实来源。它应持续维护，直到 k-Coder 的核心能力接近 cn-codex，并达到成熟编程智能体的水平。

| 项目 | 当前值 |

| 最后更新 | 2026-07-19 |

完成任务时，必须在同一次修改中更新本表和对应的任务复选框。

## 进度标记

- `[ ]` 尚未开始
- `[~]` 正在进行
- `[x]` 已完成并验证
- `[!]` 已阻塞，必须在任务旁写明原因
- 可选任务会明确标记，不会阻塞后续核心阶段

## 推进原则

各阶段必须按顺序完成。可以提前调研后续阶段，但在当前阶段通过验收门槛之前，不应合入后续阶段的生产代码。这样可以避免用大量界面和集成功能掩盖不可靠的核心运行时。

## Phase 1 - 流式对话运行时

目标：通过与供应商无关、可持久化、可取消的 `AgentRuntime` 完成真实的纯文本对话。

- [x] `P1-001` 添加 Rust 单元测试结构、测试夹具和确定性的 `FakeProvider`。
- [x] `P1-002` 定义版本化内部消息、内容、用量、Provider 事件和错误类型。
- [x] `P1-003` 把 Provider 契约改为支持取消的真实异步事件流。
- [x] `P1-004` 实现模型和 Provider 配置校验。
- [x] `P1-005` 使用操作系统凭据设施保存 API Key。
- [x] `P1-006` 实现 OpenAI-compatible Chat Completions SSE 适配器。
- [x] `P1-007` 实现纯文本 `AgentRuntime.run_turn` 循环。
- [x] `P1-008` 添加开始、文本增量、用量、完成、失败和取消事件。
- [x] `P1-009` 实现支持崩溃安全追加和恢复的 JSONL 会话事件。
- [x] `P1-010` 添加创建、列表、读取和归档会话命令。
- [x] `P1-011` 接通界面会话状态、流式消息、重试和停止控制。
- [x] `P1-012` 为分片、格式错误、错误响应和中断流添加适配器解析夹具。

验收门槛：

- 用户可以配置一个 Provider 并进行多轮纯文本对话。
- 关闭并重新打开应用后，会话和消息可以恢复。
- 点击停止能够及时取消 HTTP 流，并持久化 `turn_cancelled`。
- Provider 测试不依赖真实网络。
- API Key 不得出现在界面载荷、JSONL、SQLite 或日志中。

交付版本：`0.2.0-chat`。

- [x] `P2-001` 实现包含稳定工具名和 JSON Schema 的 `ToolRegistry`。
- [x] `P2-002` 在工具分发前完成 Schema 校验。
- [x] `P2-003` 实现工作区根目录规范化解析。
- [x] `P2-004` 拒绝绝对路径、父目录穿越、符号链接逃逸和目录联接逃逸。
- [x] `P2-005` 实现带数量限制和忽略目录规则的 `list_directory`。
- [x] `P2-006` 实现带大小限制和编码识别的 `read_file`。
- [x] `P2-007` 实现结构化工具调用和工具结果历史消息。
- [x] `P2-008` 工具返回结果后继续同一个 Turn，直到得到最终文本或达到循环上限。
- [x] `P2-009` 添加工具生命周期事件和界面工具调用记录。
- [x] `P2-010` 添加最大调用数、最大迭代数、输出限制和重复调用保护。
- [x] `P2-011` 测试无效参数、未知工具、边界路径、取消和不完整工具调用。

验收门槛：

- 模型可以只使用已注册的只读工具回答仓库问题。
- 所有只读工具都无法逃逸已选择工作区。
- 模型请求的每次工具调用都有且只有一个持久化结构化结果。
- 无限或重复工具循环会以明确的类型化原因结束。

交付版本：`0.3.0-read-agent`。

## Phase 3 - 安全编辑与审阅

目标：通过可检查、可审批的 Diff 流程完成确定性的代码修改。

- [x] `P3-001` 把严格 Patch 解析实现为独立领域模块。
- [x] `P3-002` 实现支持新增、修改、删除和移动的 `apply_patch`。
- [x] `P3-003` 添加修改前哈希，避免覆盖规划后又被改变的文件。
- [x] `P3-004` 只在明确需要完整文件替换时使用 `write_file`。
- [x] `P3-005` 在 `PolicyEngine` 中定义读取、写入、删除和外部访问风险等级。
- [x] `P3-006` 实现审批请求、通过、拒绝和超时生命周期。
- [x] `P3-007` 添加待审阅 Patch，支持按文件选择和编辑 Patch 内容。
- [x] `P3-008` 添加统一 Diff 和并排 Diff 界面。
- [x] `P3-009` 在大小限制内记录文件变更摘要及修改前后快照。
- [x] `P3-010` 为部分失败的多文件 Patch 添加回滚行为。

验收门槛：

- 不允许通过自由文本中的代码块直接修改文件。
- 审批被拒绝或超时后，文件系统不得发生改变。
- Patch 冲突必须明确报告，不能静默覆盖较新的内容。
- Diff 必须准确展示全部已应用变更。
- 写入、删除、重命名和符号链接场景都要覆盖工作区逃逸测试。

交付版本：`0.4.0-edit-agent`。

- [x] `P4-005` 实现命令状态、等待、标准输入和关闭操作。
- [x] `P4-006` 添加命令风险分类，审批原因由运行时生成。
- [x] `P4-007` 添加可配置的可复用授权，但不允许模型定义授权范围。
- [x] `P4-008` 把基于 PTY 的交互终端作为独立界面和运行时能力。

验收门槛：

- 智能体可以运行项目测试，并在同一个 Turn 中读取测试结果。
- 取消操作能够终止完整的子进程树。
- 命令不能通过修改工具参数绕过审批。
- 长时间运行的会话可以恢复查询并被显式关闭。

交付版本：`0.5.0-execution-agent`。

- [ ] `P8-006` 通过租约或冲突检测阻止多个智能体并发写入同一路径。
- [ ] `P8-007` 持久化子智能体生命周期和摘要输出。
- [ ] `P8-008` 添加运行中、阻塞、失败和完成状态的活动界面。
- [ ] `P8-009` 为并行读取、写冲突、取消和超时添加确定性测试。

验收门槛：

- 关闭子智能体必须真正取消它的模型请求和活动工具。
- 子智能体不能超出父智能体的权限和预算。
- 并行工作不能静默覆盖其他智能体的变更。
- 主智能体接收结构化结果，而不是原始隐藏对话历史。

## Phase 11 - 成熟智能体与 cn-codex 能力复核

目标：根据可靠证据而不是功能数量判断成熟度。

核心能力复核：

- [ ] 多 Provider 覆盖由真实用户需求和协议夹具驱动。
- [ ] 原生流式工具调用在所有支持的 Provider 上保持可靠。
- [ ] 读取、编辑、Patch、命令、终端、Git、计划、Skills、MCP 和子智能体达到生产可用。
- [ ] Compaction、持久化、崩溃恢复、取消、审批和审计历史达到生产可用。
- [ ] 对浏览器、记忆、Goal、机器人、移动端和插件作出明确的保留或延后决定。

成熟度标准：

- [ ] 任务评估覆盖仓库理解、针对性修改、调试、测试和重构。
- [ ] 安全评估覆盖路径穿越、符号链接、Prompt Injection、破坏性命令、密钥泄露和扩展滥用。
- [ ] 真实任务完成率、延迟、成本和故障恢复达到文档规定目标。
- [ ] 持续测量架构热点，在文件变成巨型运行时容器之前拆分。
- [ ] 项目具备稳定的扩展契约和迁移政策。

## 持续质量工作

以下工作贯穿每个阶段。

### 测试

- 对领域逻辑和解析器进行单元测试。
- 对 Provider 流和外部协议进行夹具测试。
- 对存储、文件系统边界和进程进行集成测试。
- 对关键工作流进行桌面端到端测试。
- 对智能体行为进行回归评估。

### 安全

- 默认拒绝工作区之外的访问。
- 策略决策由运行时掌控。
- 对密钥进行脱敏并使用安全凭据存储。
- 使用结构化审批和审计事件。
- 在发布节点进行依赖和威胁模型复核。

### 可观测性

- 使用带关联 ID 的结构化日志。
- 记录 Turn、Provider 调用、工具调用、审批和子智能体耗时。
- 限制诊断产物大小。
- 向用户显示不泄露密钥的明确失败原因。

### 文档

- 每个任务完成后更新本路线图。
- 把难以撤销的架构决策记录为 ADR。
- 保持运行时协议和数据位置说明为最新状态。
- 记录平台差异和已知限制。
